# L1 - High-Level Requirements

## Toromont Fleet Hub — Equipment Fleet Management & Service Portal

---

## L1-001: Fleet Dashboard
The system must provide a consolidated dashboard displaying fleet-wide KPIs, active alerts, equipment status summary, and a geographic map of equipment locations, giving fleet managers an at-a-glance operational overview.

## L1-002: Equipment Registry
The system must allow users to register, view, edit, and decommission heavy equipment assets, maintaining a complete record of specifications, ownership history, and associated documentation for each piece of equipment.

## L1-003: Service Management
The system must enable users to schedule preventive and corrective maintenance, create and track work orders through their lifecycle, and view full service history for any piece of equipment.

## L1-004: Parts Ordering
The system must provide a parts catalog with search, filtering, and ordering capabilities, allowing customers to browse available parts, add items to a cart, and submit purchase orders with delivery tracking.

## L1-005: Telemetry & Equipment Health
The system must ingest, process, and display real-time equipment telemetry data including engine hours, fuel consumption, GPS location, and diagnostic fault codes, presented through interactive charts and status indicators.

## L1-006: AI-Powered Insights
The system must provide AI-driven predictive maintenance recommendations, anomaly detection on equipment health data, and natural language search capabilities for the parts catalog.

## L1-007: User & Organization Management
The system must support multi-tenant customer organizations with role-based access control (Admin, Fleet Manager, Operator, Technician), allowing administrators to manage users, roles, and organizational settings.

## L1-008: Notifications & Alerts
The system must deliver timely notifications via in-app, email, and SMS channels for critical events including service due dates, equipment fault alerts, work order status changes, and parts order updates.

## L1-009: Reporting & Analytics
The system must generate configurable reports on fleet utilization, maintenance costs, equipment lifecycle, and operational efficiency, with export capabilities to PDF, Excel, and CSV formats.

## L1-010: Authentication & Authorization
The system must authenticate users via Microsoft Entra ID using OAuth2/OpenID Connect, issue JWT tokens for API access, and enforce role-based and claims-based authorization to ensure multi-tenant data isolation.

## L1-011: API Gateway & Integration
The system must expose all functionality through versioned RESTful APIs behind an API management gateway, supporting integration with external systems including legacy XML-based equipment data feeds.

## L1-012: Observability & Reliability
The system must implement structured logging, distributed tracing, and performance monitoring across all components, providing operational visibility into application health and enabling rapid incident diagnosis.

## L1-013: Security & Compliance
The system must implement security controls aligned with OWASP Top 10, including input validation, output encoding, CSRF protection, rate limiting, and secure data handling practices across all layers.

## L1-014: Responsive Design
The system must deliver a fully responsive user interface that adapts seamlessly across extra-small (<576px), small (≥576px), medium (≥768px), large (≥992px), and extra-large (≥1200px) viewports.

## L1-015: Event-Driven Processing
The system must process equipment telemetry events, service workflow triggers, and notification dispatches through an event-driven architecture using Azure Functions and Logic Apps for scalable, decoupled processing.
